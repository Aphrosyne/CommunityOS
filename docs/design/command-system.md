# 指令系统 Command System

> **状态：** 正式
> **版本：** v1.0
> **最后更新：** 2026-07-02

---

# 目的

本文档定义 CommunityOS 指令系统的整体设计。

指令系统是 CommunityOS 的统一命令入口，负责接收、解析和分发用户指令。

本模块不负责具体业务逻辑，仅负责将用户请求交由对应插件处理。

---

# 设计目标

指令系统的设计目标包括：

* 提供统一的命令入口。
* 降低插件之间的耦合。
* 统一命令解析行为。
* 支持插件自主注册命令。
* 为未来权限控制提供统一入口。
* 为未来帮助系统提供统一数据来源。

---

# 模块职责

指令系统负责：

* 识别命令。
* 解析命令。
* 分发命令。
* 管理命令注册。
* 返回基础错误信息。

指令系统不负责：

* 业务实现。
* 权限判断。
* 数据存储。
* 图片处理。
* 社区管理。

---

# 触发规则

指令系统的触发规则如下：

**私聊**
直接发送命令名即触发，无需额外操作。

**群聊**
必须 `@bot 命令名` 才触发，避免群聊普通对话误触指令。

**冷却**

指令冷却按等级划分，不同等级互不影响：

| 等级 | 类型 | 默认秒数 | 适用命令 |
|------|------|---------|---------|
| 0 | 查询类 | 3 | help, status |
| 1 | 会话启动类 | 5 | publish, obfuscate, decode |
| 2 | 管理类 | 10 | 后续管理命令 |

冷却在分发器层统一处理，插件无需各自实现。冷却期内静默忽略，不回复提示。

冷却时间通过 `.env` 配置：

```ini
COMMAND_COOLDOWN_L0=3
COMMAND_COOLDOWN_L1=5
COMMAND_COOLDOWN_L2=10
```

**Owner 冷却豁免**：Owner 不受指令冷却限制，用于测试和紧急处理。Admin 和 User 正常走冷却。

---

# 指令格式

无前缀，按消息首词匹配已注册命令。

例如：

```text
help

status

post
```

## 命令别名

插件注册时可声明别名，同一命令可被多个名称触发。

```python
register("help", handle_help, description="显示帮助信息", aliases=["帮助"])
```

`help` 和 `帮助` 均触发同一处理器。

---

# 工作流程

命令处理流程如下：

```text
用户发送消息

↓

指令系统识别指令

↓

解析命令名

↓

查找已注册命令

↓

调用对应插件处理器

↓

插件返回结果

↓

发送回复
```

指令系统不关心插件内部如何实现。

---

# Plugin → Service 分层

所有指令遵循统一的分层模式：

```text
用户

↓

指令系统（识别 / 分发）

↓

Plugin（业务组装，决定展示什么）

↓

Service（数据来源，决定数据怎么做）
```

- **Plugin** 负责：展示逻辑、格式化回复、调取多个 Service。
- **Service** 负责：数据来源，不关心谁在用。
- **指令系统** 负责：识别、解析、分发，不参与业务。

# 命令注册

所有插件均可注册自己的命令。

插件通过 `services/command.py` 提供的 API 注册：

```python
from services.command import register

async def handle_status(bot, event):
    reply = f"运行时间：{get_uptime()}"
    await bot.send(event, reply)

register("status", handle_status, description="查看系统运行状态")
```

注册在模块导入时完成，无需手动初始化。

指令系统不维护具体业务。插件负责声明自己支持哪些命令。

---

# 命令分发

当消息被识别为指令后，分发器：

1. 提取命令名。
2. 检查用户冷却。
3. 查注册表。
4. 找到则调用 handler。
5. 未找到则回复「未知命令，输入 /help 查看可用命令」。

分发器统一处理所有进入指令系统的消息，插件只接收自己注册的命令。

---

# 未知命令

当用户输入不存在的命令时，指令系统应：

* 返回统一提示。
* 不抛出异常。
* 不影响其他插件运行。

# 已实现（v1.0）

| 组件 | 文件 |
|------|------|
| 命令注册中心 | `services/command.py` |
| 命令分发器 | `plugins/command_dispatcher.py` |
| `/help` | `plugins/help.py` |
| `/status` | `plugins/status.py` |
| 权限服务 | `services/permission.py` |

配置项：

```ini
COMMAND_COOLDOWN_L0=3     # 查询类冷却（秒）
COMMAND_COOLDOWN_L1=5     # 会话启动类冷却（秒）
COMMAND_COOLDOWN_L2=10    # 管理类冷却（秒）
OWNER=0                   # 机器人所有者
ADMINS=                   # Bot 管理员
```

特性：无指令前缀、命令别名、首词匹配、按等级冷却、权限检查、Owner 冷却豁免。

# 总结

指令系统是 CommunityOS 的统一命令入口。

它负责识别、解析和分发命令，为所有插件提供一致的调用方式。

其目标不是实现业务，而是建立统一的交互入口，使各插件保持独立、可扩展和易于维护。
